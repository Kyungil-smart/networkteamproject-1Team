# 로비 구현 작업 사항
## 04-30 강사님 코드 RnD
### 코드 관계 분석후 프로젝트에 사용할것들 📁Project/Scripts 폴더에 추가
```
- Core
	- SceneLoader
- Model/Title
	- AuthService  : 초기화/익명 로그인 (static)
- Model/Lobby
	- LobbyManager  : 핵심코드. 4가지 경로로 세션 진입시 Lobby + Relay + NGO 셋업
	- LobbySettings  : 플레이어 수 설정 (SO)
	- LobbyConstants  : 상수 필드 보관 (static)
- Model/InGame
	- GameSessionManager : 게임 씬에서 플레이어 합류 및 게임 시작/종료를 처리
- UI/Title
	- TitleUI        : 타이틀 UI. 이름 입력, 로그인 후 Lobby 씬으로 이동
- UI/Looby
	- LobbyListUI      : 로비 핵심 UI. 세션 목록 + 방 생성/코드참여 팝업, 빠른참여/새로고침 컨트롤
	- LobbyEntryUI     : 세션 목록의 한 줄(방 하나)을 표현, 'LobbyEntry'프리펩에 부착 (LobbyListUI 에서 사용)
	- RoomUI            : 방 핵심 UI. 'RoomPanel'에 부착. 세션이 업데이트 될때마다 Refresh() 실행
	- RoomPlayerSlotUI  : 참가자 정보 UI. 'RoomPanel/PlayerSlotsContainer'에 배치된 slot프리펩들에 사용(현제 4개 배치중)
	- CreateRoomDialogUI : 방 생성 팝업, 방 이름 결정후 LobbyManager.Instance.CreateSessionAsync 호출
	- JoinByCodeDialogUI : 방 코드 참여 팝업, 코드 입력후 LobbyManager.Instance.JoinSessionByCodeAsync 호출
- UI/InGame
	- GameWaitingUI : 인게임 합류 대기 UI.
```
- 흐름 정리
	- 타이틀: AuthService 익명 로그인 → 이름 입력후 로그인 → Lobby 씬 진입
	- 로비: LobbyListUI 방목록 표시 → CreateRoomDialogUI 방생성 → LobbyEntryUI 방표현 → 방 참여/종료시 LobbyListUI.ShowLobbyListPanel(bool)에서 로비패널/룸패널 전환
	- 룸: RoomUI 룸정보 표시 → RoomPlayerSlotUI 참가자 정보 표현 → 모든 플레이어 Ready면 TryStartGameAsHostAsync() 호출하여 게임 시작
	- 인게임: GameSessionManager 게임 씬 진입 이벤트 처리 → GameWaitingUI에서 플레이어 합류 대기 → 모든 플레이어 합류하면 게임 시작
