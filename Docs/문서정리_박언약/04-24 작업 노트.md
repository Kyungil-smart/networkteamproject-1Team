# 04-24 작성 코드

## PlayerSpawnManager
- 기본적으로 NetworkManager의 OnLoadEventCompleted 이벤트에 SpawnAllPlayers 등록하지만 테스트용 호출 가능
- `SpawnAllPlayers`에서 로드 완료한 플레이어를 `A(시민)` / `B(마피아)` 팀으로 나눈다.
- 딕셔너리 assignTeams에 플레이어별 배정된 팀을 `<클라이언트ID, 팀>`형태로 저장(`<ulong, TeamType>`) 
- `B` 팀 인원은 `_targetTeamBCount` int로 고정한다.
### A/B 그룹 분리 동작 방식
1. 스폰시 `clientsCompleted` 목록을 복사한다.
2. 복사한 목록을 랜덤 셔플한다.
3. 셔플된 순서에서 앞의 `_targetTeamBCount`명만 `B`로 배정한다.
4. 나머지 인원은 모두 `A`로 배정한다.
### 참고
- 외부에서는 `IsTeamB`로 클라이언트의 배정 결과를 조회할 수 있다.

## EditorNetworkTest
- 간단하게 테스하기위한 에디터 스크립트입니다 
- Settings/Editor 폴더에 프리펩을 두었습니다
- PlayerSpawnManager을 참조 (Reset을 실행하면 씬에서 찾아서 할당)
### 테스트 동작 방식
- F1: StartHost
- F2: StartClient
- F3: SpawnAllPlayers (맵 씬에서 테스트용, 로비에서 씬으로 이동시에는 자동으로 실행됨)
